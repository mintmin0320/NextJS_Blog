## 👨🏻‍💻 프로젝트 - 관리자/주문목록 페이지 만들기
아직 서버를 담당하는 친구가 개인 사정으로 개발이 미뤄지고 있는 관계로 HTML/CSS와 더미 데이터를 활용해 최대한 개발을 진행해 나가고 있다. 비록 CSS 및 디자인 실력이 부족해 퀄리티는 낮지만, 최대한 심플하고 기능 구현에 목적을 두려고 생각했다.

오늘의 목표 관리자/주문목록 페이지는 프로젝트 메인 기능이라고 볼 수 있을 정도로 중요한 기능이기 때문에 시간이 상당히 오래 걸렸다. 관리자 페이지는 사용자들이 접근하는 페이지가 아니기 때문에 디자인적인 부분을 신경 쓸 필요가 없다고 생각했고 최대한 효율성을 고려해 개발을 진행했다.

![](https://velog.velcdn.com/images/mintmin0320/post/6e2333f7-9015-47a8-a823-9a2420f9ba68/image.png)

좌측에 사이드바를 제외하고 크게 주문목록과 주문을 상세하게 확인 부분을 상하 영역으로 나누었다.
초기 구상안은 주문 목록을 column 정렬할 계획이었으나 사용자의 모든 데이터를 담기에는 공간이 협소하고 가독성도 떨어진다고 판단했다. 그래서 grid를 사용해 정렬했고, 상품 주문 시 입력하는 과목명과 활용 계획만 목록에 표시했다. 목록을 선택하면 하단에 사용자의 상세 정보와 상세 주문 목록이 표시되고 그 외에도 주문 거절(취소) 및 승인 버튼을 배치되어 있다.

### 🔎 문제
오늘 가장 많은 시간이 소모됐던 부분은 하단(주문 상세 처리)부분이다. 

![](https://velog.velcdn.com/images/mintmin0320/post/49f4c983-4cd7-464c-bba0-8f48af684e5f/image.png)


![](https://velog.velcdn.com/images/mintmin0320/post/282d1979-f29b-4146-9621-ed0e5adcced6/image.png)

1, 2번과 3,4번 각각 좌우로 크게 영역을 나누고 그 안에서도 4부분으로 영역을 나누었는데, 각 영역의 border를 맞추는 부분에서 많은 시간이 소모되었다.
처음에는 영역의 border 고려 없이 만들 계획이었으나 1번과 3번의 선의 간격이 의도치 않게 맞춰지면서 결국 이쁘게 맞추기로 결정했다. 항상 CSS를 만들 때 1px이라도 어긋나면 참을 수 없다..

![](https://velog.velcdn.com/images/mintmin0320/post/1e203d7b-c56f-428b-97df-e2c074865db3/image.png)(불편하다..)

<br/>

처음 구상한 계획은 스크롤 없이 현재 화면을 모두 확인할 수 있도록 height: 100vh을 부여했다. 딱 맞게 height를 부여하고 있었으나 border 1px로 인해 스크롤과 영역들이 어긋나는 상황이 발생하기 시작했다.
(1px씩 어긋나니 잘 보이지도 않아서 더 열받는다. 🤯)
다행히도 최근에 calc() 함수를 알게 됐던 기억으로 width와 height에 border 1px을 빼주면서 다행히 딱 맞게 배치가 완성됐다.

> 📌 calc() - 속성의 값을 계신식으로 지정 가능
- %로 지정된 값을 계산할 때 어려움을 해결할 수 있다.
ex. width: calc(100% - 80px);
